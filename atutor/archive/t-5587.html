    Re: Security and themes problem

    You are right about security on the content/ directory being writable and Web accessible, though it is minor unless your content needs to be hidden. This is explained in Step 4 of the installation, recommending you locate the directory outside your Web root directory. Various checks and permissions keep the possibility of malicious behaviour to a minimum if you decide to leave your content directory web accessible.

    Details on the final installation screen that explain various security strategies (delete the install directory, reset permissions on the config.inc.php file with "chmod a-w config.inc.php") seem to have been lost. We'll put them back in.

    I can not reproduce the theme problem. I can't say why the problem occured, though the course ID is missing from the paths in the error messages above, so the session running (or not) may be the problem. This is likely a browser issue. What happens if you restart your browser, or logout and login again? ...try a different browser? etc.
