    I was not able to import your test package but was able to create another and import it. When I imported your package, I don't get a blank screen however, but rather an ATutor error message that the package is not a QTI 1.2.1 compatible package.

    There's a a couple possibilities that might cause this. If content was pasted in from a word processor, MS Word and others, often invisible foreign characters are introduced that will break packages when they are exported.

    The other possibility is the content was created in a character set other than UTF-8. This is problematic when Spanish special characters are used (e.g. ).

    You could experiment by exporting individual questions. I expect you will be able to export/import some questions but not others. Those you cannot import recreate them, retyping them manually, then exporting again.
