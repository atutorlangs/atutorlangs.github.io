  • 2005-07-25 09:54:13

    Imports for me

    I tried importing into the atutor.ca demo site, and it seems to have imported ok, though the javascript is missing, which is normal since ATutor does not currently import information in the head of HTML documents (we're going to change this). All the images etc. are imported properly, and show up in the content.

    There are some problems with the Flash. Inside ATutor or outside on their own, they do not play. It could be the way WebCT exports them. It could also be that required files for the Flash are missing, if they originally existed with the Flash, but are not explicitely referenced as a PARAM.

    If exported from ATutor, which builds the content package, rather than copying as WebCT, and some other systems do, the package will be missing resource files that are not explicitly referenced. Since Flash and other such content exists in binary form, ATutor can not access the binary code to find the resource files referenced from within the Flash itself. This is a general issue with content packages that has not yet been addressed in the IMS specifications. Not all content packages will work in all systems unless resource files are explicitely referenced.

    With some minor editing though, I'm sure you could make your imported package function properly, and when exported after explicitly referencing resource files in the HTML PARAM element, the content will be fully interoperable with other systems.
