    Re: Setting scope on atutor.ca forum searches

    Greg, Hope it was clear that I meant setting the search scope to a sub-branch (or sub-branches) of the atutor.ca site rather than outside of (beyond?) it.

    The full-site search with refinement on results and google parameters is sufficient once you are familiar with the site. As a newb, I was looking for a clean way to check the reported bugs before opening a new Bug Report thread. Including the filter: [quote] site:www.atutor.ca/atutor/mantis/view.php OR site:www.atutor.ca/view/3 OR site:www.atutor.ca/development/bugs[/quote]
    works, but is a bit cumbersome to recall or cut & paste when needed.

    If you don't apply this filter for something like an eXe package related problem, you can't be certain that reported bugs don't exist among the items below the top 20 that are returned - regardless of how you refine your boolean search phrase. With the filter applied, "eXe" returns only 3 bug entries.

    Not urgent, but it would be helpful to have something like a "Search Bugs" link on the site that would simply append or apply this site filter so that the returned items would all be bug report related.

    Thanks!
