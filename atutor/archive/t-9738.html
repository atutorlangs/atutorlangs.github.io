    Re: LCMS selection: various questions

    1. Scalability and Performance.
    I read some notes on this in the fora, there I understand scaling-up means faster hardware. What would it take to support 25.000 students and 10.000 staff members (We're Unix based)? What's the experience here? What tooling is available to ATutor system administrators to handle that number of users?

    [green]ATutor is easily the most resource efficient LMS available. It will have
    no trouble handling 35,000 users on moderate hardware. There are modules
    available for LDAP, and custom modules would be fairly easy to create to
    link into other directory authentication systems if necessary. It is
    also possible to import a confirmation list that only allows eligible
    students to register. [/green]

    2. What API's are available? (Apart from SCORM) This is important, for instance, for integration of ATutor with the local repository.

    [green]Modules API. ATutor is easily extended with modules. See the module
    development documentation.
    http://www.atutor.ca/development/documentation/modules.html
    http://www.atutor.ca/development/documentation/guidelines.html [/green]

    3. Important for swift cooperation between multiple parties is delegated user management. Is this supported? E.g., are regular instructors allowed to give access to external people?

    [green]Multple administrators are possible, with the ability to assign user
    management privileges one or several. Instructors can also add users to
    their own courses. [/green]

    4. It appears not to be possible for instructors and course attendants to upload a set of files. It's one by one. Correct?

    [green]Correct, though one could zip together a collection of files and unzip
    them on the server [/green]

    5. Is there a Review/feedback-mechanism of content available?

    [green] Not specifically associated with content. But, the forums would work for
    this purpose. The File Storage Utility does provide commenting on draft
    documents[/green]
    6. Is gathering, managing and sharing of Bookmarks available?

    [green]Yes. The Links tool [/green]
