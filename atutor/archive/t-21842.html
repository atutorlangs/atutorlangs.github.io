    Re: Auto-enrollment issue (can't create new student)

    We see this type of problem occur occasionally when systems are upgraded. ATutor 1.6.3 was created before PHP 5.2.11, so it is possible something changed in PHP that's causing the problem. Which means upgrading may be your best solution to the problem. More effort may go into figuring out what's causing the problem, than it would take to do an upgrade.

    Upgrades are not difficult, though recreating the theme will need some effort. Perhaps one of the themes for ATutor 2 will be a sufficient replacement.

    http://atutor.ca/atutor/docs/installation.php#upgrade

    Sorry we can't help more. This is the type of problem that could take a lot of time to investigate.

    If you want to investigate yourself, its the "members" table new accounts get added to.

    Also try using the default them when creating a user, to see if it Corporate theme is the problem.

    ID and PIN are not the same as user/password.
    This is a System Preference, "authenticate against master list" Turn off the preference and the problem should go away.

    Try to be as exact as possible when you're describing problems you might be having.
