    Re: error in language on Theme Mobile

    Language is separate from themes. Both Blumin and mobile would use the same language, unless there happens to be some hard coded language in the theme that did not get translated.

    You can edit language using Admin>SystemPreferences>Languages>Editor

    or you can turn on Translation, and edit from there.

    See "Languages" in the admin handbook for details.

    I've posted patches 0001 to 0003 for ATutor 2.2.2, which fixes the hard coded "Don't have an account"(and an issue with the Patcher)

    As for "Off" and "Topics In" in the mobile themes, these terms are in the language for 2.2.2. My guess is the language has been cached from your old installation.
