    You do need to create a content directory if the original was inside your old installation. If the old one was outside the ATutor directory in the old installation, you won't need to create a new content directory.

    As for stopping at step 6, that is when content is copied from the old install to the new one if the content directory is inside the ATutor directory. Perhaps the permissions on the directory are not set to make it writable, or there is a lot of content in the old installation, and it is taking a long time to copy over.

  • 2009-01-21 10:38:01

    Re: upgrading from atutor 1.5.2 to 1.6.1 stops in step 6

    Mixed English/Farsi course will break when you convert the system to UTF-8. The converter has to know what character set to start with, then converts. If you have english mixed with Farsi, the converter will attempt to convert the Farsi thinking it is English.

    Into the future this won't be a problem, when everything is in utf-8. This is part of the reason we upgrade to utf-8, so mixed language course could be created.

    The only solution may be to recreate the Farsi content. Also when you convert courses that are all written in Farsi, make sure the course itself is set to Farsi as its default language, in Course Preferences.

    Re: upgrading from atutor 1.5.2 to 1.6.1 stops in step 6

    I had this problem before and finally I did the conversion manually.
    if you have a backup of your database, first restore it and then export all your database in a .sql file. then use a powerful editor like ultraEdit to change the conversion.
    if your old conversion is windows-1256 and now you want to convert it to utf8 do the following steps:
    1. open your .sql (windows-1256) file in ultraedit and do these conversions:
    a. file -> conversions -> ASCII to Unicode
    b. file -> conversions -> Unicode/ASCII/utf8 to utf8(ASCII editing)
    all of your data (English or farsi) is now in utf8
    2. save you file and create a new database with default character set utf8:
    create database 'db-name' default character set utf8 collate 'utf8_persian_ci';
    3. import .sql file into new database.
    4. do the upgrade again but with the new database and skip conversion.
    5. after upgrade if you still can not read farsi characters, modify ATutor\include\lib\mysql_connect.inc.php and add mysql_query("SET NAMES 'utf8'"); at end of the page berfor }
    hope this can help you.
