    Content associated files only

    How would your students access the authorware content? I'm assuming a link in a content page allows them to download the file. If you export the page with the link in it, the packager will find the authorware content and create a content package. Exporting the entire content directory would be undesirable in most cases, particularly if you are exporting smaller units within a course as content packages.

    One thing ATutor content export can not deal with (nor can other utilities of this sort) is internally defined files accessed via binary content. You might try creating hidden links within the page containing the link to your authorware content, that link to the files it calls.
