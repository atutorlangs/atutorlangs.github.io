    Re: but it necessary

    Obviously HTML can be sent through Outlook, and most other mail applications. But sending html from them ( desktop applications) and from a ATutor (a Web application) are two very different things. Allowing HTML in a Web application opens up a whole variety of potential problems.

    In the context of an LMS, I can see no good reason why we should allow HTML in mail messages. For the amount of effort that would need to go into allowing "pretty email" in a Web app, and the insignificant pedagogical value of pretty email, I'd give up on trying to have it introduced in ATutor.
