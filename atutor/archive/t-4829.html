    Editing files using the File manager

    One of our modules consists of a .swf (Flash) file. This .swf file uses many external images and corresponding descriptions in .txt files. The links to these images and text files are hard-coded in the .swf file.

    When I open the File Manager for this module, only the index.html file is in the main directory, and all the other files (.gif, .txt and .swf) are in a single subdirectory below that.

    The problem I'm having is that when you edit a .txt file in the subdirectory, the modified version is being saved in the main directory. It doesn't overwrite the original .txt file in the subdirectory. Is there a way to change this behavior?

    (ATutor 1.4.3 on Red Hat Linux)

    Thank you.

    --John


  • 2005-06-13 17:51:32

    Fixed in 1.5

    This problem has been fixed in ATutor 1.5. For now you'll need to use the Move feature in the File Manager to put the file where it belongs.
