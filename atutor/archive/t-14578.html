  • 2008-08-27 03:19:12

    Re: alteration fail in upgrade from 1.5 to 1.6.1

    I have this problem during upgrade from 1.5.5 to 1.6.1 with farsi contents.
    I think the best way is to change the encoding of the data manually. Do the following steps:
    1.use phpMyAmin to export your database data
    2.open the dump file in UltraEdit editor and change the encoding by File->Conversion ->"utf8 to Unicode" convert it again to "Unicode to utf8( ASCII editing)" and save the dump file.
    3.open the dump file with notepad to sure about the true conversion. Farsi characters should be readable in notepad.
    4. empty your tables and import the dump file into the database.
    Note: be sure that your database default "character set" set to utf8.
    I hope it will be useful.
