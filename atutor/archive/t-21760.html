    I met with a complaint which is exactly same with mine mentioned above on the URL "http://www.joomlablogger.net/joomla-tips/joomla-extension-tips/tinymce-respecting-settings/"
    ----
    The problem one Swedish user had was as follows: He inserted a picture, entered settings for alignment, horizontal space etc. and everything looked good. Happily he saved the article. When he looked at the article again, all the formatting was gone. What happened?

    This is actually very easy to fix. TinyMCE has a setting for stripping code from the text when saving. It's called "Code Cleanup on save".

    Go to Plugin Manager and Editor - TinyMCE 2.0. You'll find the setting under Plugin Parameters. The setting should be set to "Never".

    Re: Probem with alignment for pictures

    tinyMCE settings are stored in include/lib/tinymce.inc.php

    I think you can add "cleanup: false" or something into the settings there to manually turn it off. I haven't checked what the exact code should be like on tinyMCE. You can try searching for it on Google or tinyMCE and add it yourself.

    Another note is on the 4th row in our tinyMCE, the second last button from the right has a button called "cleanup messy code". I think that is the function you are looking for.
