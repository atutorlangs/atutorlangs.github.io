    Writable import dir

    Make sure the content/import/ directory is recursively writable. You must use the "-R" switch when setting permissions on the content directory.

    chmod -R a+rwx content/import/


    Unfortunately we don't have an IIS server available on which to reproduce this problem. If possible the best server combination is Apache 1.3 using PHP as an Apache module. There have been other problems reported when using the CGI version on Windows.
