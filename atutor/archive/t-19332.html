    Re: problem with downloading files from file storage and file manager

    Mention this to your Internet provider, if this is something that started recently. It may be related to a recent PHP upgrade, or other changes to the server.

    To confirm, permission on the content/file_storage/ directory, and its sub directories, should be set to rwxr-xr-x (755) and owned by nobody:nobody (assuming that's your webserver user). Files themselves rw-r-r (644).

    This should all be setup automatic when the content directory gets created during installation. Something else may be going on if you have to reset permissions.
